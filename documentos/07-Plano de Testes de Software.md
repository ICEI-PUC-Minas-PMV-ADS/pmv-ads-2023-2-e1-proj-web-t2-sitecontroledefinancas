Os requisitos para realização dos testes de software são:

Site publicado na internet;
Navegador da internet: Chrome, Firefox ou Edge.

Os testes funcionais a serem realizados na aplicação são descritos a seguir.

Preencher o campo de senha com uma senha inválida e Clicar no botão "Entrar". **Passo 3**: Clicar no botão “Cadastrar” **Passo 4**: Verificar se todos os elementos (botões, campos, links) estão funcionando corretamente e são clicáveis. **Passo 5**: Testar a responsividade da página em diferentes dispositivos. |
| Critérios de êxito | O redirecionamento para a página principal é feito corretamente após preencher o campo e-mail e senha corretamente e clicar no botão “Entrar”. A mensagem de erro "Dados inválidos" é exibida quando os dados informados não correspondem aos exigidos no campo e-mail e senha A página de “Cadastro” é carregada corretamente quando clicar no botão “Cadastro”.A pagina carrega corretamente em diferentes dispositivos |

|**CT - 03** | **Teste da página "Home"**|
|:---|:---|

Passo 3: Verificar a exibição do gráfico com informações das despesas e receitas. Passo 4: Utilizar a opção de selecionar o mês, o ano, filtrar e limpar filtro. A pagina carrega corretamente em diferentes dispositivos |

|**CT - 04** | **Teste da página "Despesas"**|
|:---|:---|
| Requisitos Associados | RF-4, RF-6, RF-7, RF-12 |
| Objetivo do Teste | O objetivo dos testes para a página de Despesas é validar a funcionalidade e precisão dos elementos presentes nessa seção. Isso inclui verificar se o botão "Nova Despesa" abre corretamente uma nova área ou aba para registrar despesas, confirmar a visualização precisa das despesas pagas, pendentes e o total de despesas, garantir a listagem completa e correta de todas as despesas e verificar se os botões do menu lateral redirecionam adequadamente para suas respectivas páginas associadas   |
| Passos |  Passo 1: Clicar nos botões "Início", "Receitas", "Meu Perfil" e "Sair" presentes no menu lateral. Passo 2: Verificar a listagem completa de todas as despesas na página, clicar no botão “Excluir” que fica na coluna Deletar, clicar no botão “Pagar” quando as despesa estiver  pendente. Passo 3: Clicar no botão "Nova Despesa" e verificar se uma nova aba ou seção é aberta para o registro de uma nova despesa.Passo 4: Preencher os campos de Valor, Data, Descrição, e abrir a aba suspensa para selecionar a categoria. Passo 5:    Abrir a aba suspensa para selecionar a categoria. Escolher a opção "Outros" para cadastrar uma nova categoria. Passo 6: Marcar a despesa como paga ou não e se a despesa foi paga com Cartão de Credito Passo 7: Cadastrar uma nova despesa com um valor específico, com descrição, data, categoria e se ela esta pagou ou não e depois clicar no botão “Cadastrar” Passo 8: Confirmação da exibição correta e atualizada das despesas pagas, pendentes e o cálculo preciso do total de despesas.  Passo 9: Simular despesas que atinjam 80% das receitas. Passo 10: Testar a responsividade da página em diferentes dispositivos |
| Critérios de êxito |  Passo 1: Os botões do menu lateral redirecionam adequadamente para suas respectivas páginas associadas: "Início" para a página inicial, "Receitas" para a seção de receitas, "Meu Perfil" para a área de perfil do usuário e "Sair" para página login. Passo 2: Todas as despesas estão sendo exibidas corretamente na lista, com informações precisas, ao clicar no botão “Excluir” a despesa é excluída, ao clicar no botão “Pagar” e despesa mudo de pendente para “Paga”Passo 3: Ao clicar no botão "Nova Despesa", uma nova área ou aba é aberta para o registro de despesas. Passo 4: É possível inserir informações nos campos de Valor, Data e Descrição, verificando se a entrada é aceita corretamente, e verificar a aba suspensa de escolha de categoria preexiste é possível. Passo 5: É possível selecionar uma categoria existente e também cadastrar uma nova categoria quando a opção "Outros" é escolhida. Passo 6: É possível marcar e desmarcar a despesa como paga ou se ela foi paga com Cartão de Credito, observando se essa modificação reflete corretamente nos valores totais de despesas pagas e pendentes e cartão de credito. Passo7: Os valores de despesas pagas, despesas restantes e total de despesas são atualizados corretamente após o cadastro da nova despesa. Confirmar se o valor adicionado na nova despesa é refletido de forma precisa no gráfico da tela inicial, mostrando a atualização dos valores de acordo com a despesa cadastrada. Passo 8: Exibição correta e atualizada das despesas pagas, pendentes e o cálculo preciso do total de despesas.Passo 9: Ao atingir 80% das despesas em relação às receitas, uma mensagem informativa é exibida na página inicial. Passo 10: A página carrega corretamente em diferentes dispositivos   |
